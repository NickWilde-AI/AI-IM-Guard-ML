# 当前限制与未来路线图

本文档整合项目当前的已知边界和下一阶段演进方向，便于面试答辩时主动暴露工程判断力。

## 1. 当前边界（诚实声明）

### 数据层

| 限制 | 影响 | 缓解措施 |
| --- | --- | --- |
| 无真实 IM 私聊业务数据 | 模型泛化能力未验证 | 接入 XGuard 200K 公开安全底座 + HeuristicJudge 兜底 |
| 公开数据只能标注 low/mid_risk | 无法训练 ban_account 决策 | 字段级 loss mask 防止标签污染 |
| 无人工审核回流样本 | 缺乏 hard case 迭代能力 | 预留 refinement 模块和 committee 投票接口 |

### 模型层

| 限制 | 影响 | 缓解措施 |
| --- | --- | --- |
| 无真实 Qwen checkpoint | TransformersJudge 无法直接运行 | 提供完整训练入口，本地可复现 SFT |
| 无线上 A/B 平台 | 灰度对比仅在配置和设计层面 | rollout.yaml + canary 最小实现 |
| 评测指标来自设计文档 | 指标非实测 | experiment_tracking 模块可记录真实跑分 |

### 服务层

| 限制 | 影响 | 缓解措施 |
| --- | --- | --- |
| SQLite 审计为单实例 | 多副本无法聚合查询 | 支持 JSONL + SQLite 双后端，预留 PostgreSQL 升级路径 |
| 限流为进程内滑窗 | 多实例无全局限流 | 配置文档明确生产应接入网关限流 |
| 压测为单客户端顺序请求 | 无法验证并发性能 | 新增 concurrent 压测模式（本次补齐） |
| 无企业密钥轮换 | token 泄露后需手动更新 | 支持 SHA-256 hash token，降低明文泄露风险 |

### 部署层

| 限制 | 影响 | 缓解措施 |
| --- | --- | --- |
| K8s 模板未对接真实集群 | 灰度自动化未验证 | 提供完整 Deployment/Service/PVC/ConfigMap |
| 无 GitOps 或 ArgoCD 集成 | 发布流程依赖手动 | CI pipeline 已覆盖质量门禁 |
| 无多实例 soak test | 长时间稳定性未验证 | benchmark_api 提供基线门禁 |

## 2. 路线图

### Phase 1：面试展示就绪（当前）

- [x] 完整 6 层架构代码实现
- [x] 公开数据接入与保守标签映射
- [x] API 服务化 + 鉴权 + 审计 + 监控
- [x] Docker/K8s/vLLM 部署模板
- [x] 137 单元测试 + CI 门禁
- [x] 模拟看板 + 实时数据大盘
- [x] 面试文档全套（playbook、QA、简历叙事）

### Phase 2：内部试用（预计 2-4 周）

- [ ] 接入真实 IM 私聊脱敏样本（500-1000 条人工标注）
- [ ] 完成首轮 Qwen SFT 训练，产出真实 checkpoint
- [ ] 跑通离线评测，验证 F1 > 0.75 基线
- [ ] SQLite 审计升级为 PostgreSQL
- [ ] 增加多并发压测和 soak test（>1h）
- [ ] 对接企业密钥管理（Vault / KMS）

### Phase 3：灰度上线（预计 4-8 周）

- [ ] 人工复核闭环：reviewer 提交 correction → 模型回流
- [ ] 真实集群灰度：canary 5% → 20% → 100%
- [ ] 网关级限流和租户隔离
- [ ] Prometheus + Alertmanager 真实告警链路
- [ ] 线上 A/B 平台对接
- [ ] 合规审计留存（90 天 + 归档）

### Phase 4：全量生产（预计 8-12 周）

- [ ] 多模型 ensemble 和 committee 投票
- [ ] 用户申诉链路和标签修正回流
- [ ] 跨业务复用（群聊、直播弹幕、评论区）
- [ ] 模型蒸馏：27B → 7B，降低推理成本
- [ ] 联邦学习探索：多租户数据不出域

## 3. 面试表述建议

> 面试官问："这个项目能直接上线吗？"
>
> 回答：不能直接上线，但这是设计意图。项目定位是生产化工程方案验证，证明我能从数据、训练、服务、监控、治理全链路设计一个企业级系统。真实上线还需要三件事：业务私有数据、人审闭环、和平台级部署治理。这三件不是技术难题，而是需要团队和资源配合的工程落地问题。

## 4. 与同类项目的差异

| 对比维度 | 普通 NLP 分类项目 | 本项目 |
| --- | --- | --- |
| 输出 | 单标签 safe/unsafe | 多字段结构化（risk_level + judgment + handling + route + action） |
| 数据 | 单一来源 | 多源融合 + 字段级 loss mask + 质量审计 |
| 服务化 | Flask demo | FastAPI + 鉴权 + 限流 + 审计 + 版本追踪 |
| 监控 | 无 | Prometheus + 滑动窗口 + drift + 告警 |
| 部署 | 无 | Docker + K8s + vLLM + 灰度配置 |
| 治理 | 无 | 模型注册表 + 审批 + 回滚 + 人审 |
