# Rubric 与标注规范

这份文档解决两个问题：业务违规体系怎么拆，标注员如何稳定给出 `risk_level / final_judgment / handling_suggestion`。面试里如果被问“你的标签从哪来、怎么保证一致性”，优先讲这份规范。

## 1. 标注输入

每条样本必须给标注员看到三类证据：

- `audit_scene`：聊天类型、亲密度、行为摘要、礼物金额、进房、关注、登录等。
- `chat_evidence_list`：关键聊天消息、时间、risk point。
- `behavior_abnormal_list`：异常行为类型、描述和证据。

标注员不能只看聊天文本，也不能只看行为阈值；必须判断语义和行为是否构成同一条证据链。

## 2. 标注输出

每条样本标注六个字段：

- `topic`：11 类主题之一，安全样本为 `无主题`。
- `risk_level`：`low_risk / mid_risk / high_risk`。
- `final_judgment`：`exist_violation / not_exist_violation`。
- `handling_suggestion`：`ignore / warning / limit_account / ban_account`。
- `correlation_analysis`：语义和行为如何关联。
- `judgment_basis`：引用证据说明为什么这样判。

## 3. 总体判定原则

`low_risk`：

- 日常闲聊、普通社交、正常消费咨询。
- 有轻微擦边但证据不足。
- 行为正常，且聊天语义没有明确违规意图。
- 通常对应 `not_exist_violation + ignore`。

`mid_risk`：

- 语义或行为存在违规倾向，但证据链不完整。
- 首次出现低烈度违规。
- 文本暗示较强但行为不印证，或行为异常较强但文本很隐晦。
- 通常对应 `exist_violation + warning/limit_account`。

`high_risk`：

- 语义和行为相互印证，证据链完整。
- 明确禁止类目，如诈骗、违禁品、未成年严重诱导。
- 存在批量投递、大额消费、外部交易、明确金额或交付安排。
- 通常对应 `exist_violation + limit_account/ban_account`。

## 4. 处置档位原则

`ignore`：无违规或风险极低。

`warning`：低中风险、首次出现、证据不完整，但需要站内提醒。

`limit_account`：中高风险，已有行为或语义印证，但直接封号证据不足。

`ban_account`：高风险且证据链完整。生产上模型只给 ban 建议，最终必须人审复核。

## 5. 11 类主题 rubric

详细配置见 [configs/rubrics.yaml](../configs/rubrics.yaml)。面试讲解时不需要背 47 个子主题，但要能举 2 到 3 个典型例子。

### 5.1 代刷/包榜

子主题：约定金额冲榜、包榜代刷服务、返利互刷、暗语冲榜。

- low：只讨论榜单玩法或正常送礼。
- mid：暗示“老规矩”“冲一波”“互送返利”，但缺金额、对象或行为印证。
- high：明确金额、榜单位次、时间，并出现短时大额打赏。

### 5.2 诈骗引流

子主题：稳赚项目、投资返利、外部私聊引流、批量投递。

- low：普通联系方式交换，没有收益承诺。
- mid：诱导加外部账号，暗示收益或批量投递。
- high：明确本金、返利、押金、收益承诺或索取敏感信息。

### 5.3 私下交易

子主题：平台外结算、折扣交易、绕过抽成、私下收款。

- low：平台内正常消费咨询。
- mid：试探加微信谈折扣或平台外结算。
- high：明确收款方式、金额、交付安排。

### 5.4 色情诱导

子主题：露骨邀约、付费色情引流、擦边暗示、外部私密内容交易。

- low：普通暧昧或玩笑。
- mid：擦边暗示、私密内容试探、外部联系方式。
- high：明确色情服务、裸露内容交易或付费私密引流。

### 5.5 未成年保护

子主题：未成年身份试探、诱导私聊、不当内容暴露、金钱诱导。

- low：普通交流，无未成年线索。
- mid：出现未成年身份线索，并伴随私聊或不当话题试探。
- high：针对未成年进行色情、金钱、外部引流或线下接触诱导。

其他主题可按同一原则讲：先看语义强度，再看行为印证，再看业务处置代价。

## 6. 标注一致性流程

推荐流程：

1. 每条样本由 3 名标注员独立标注。
2. 完全一致直接采纳。
3. 两人一致取多数票。
4. 三人分歧进入仲裁。
5. 每周抽样复盘分歧样本，更新 rubric。

一致性指标：

- `Fleiss Kappa`：多标注员一致性。
- `ordinal Krippendorff alpha`：适合 risk_level 这种有序等级。
- `Cohen's Kappa`：成对一致性校验。

## 7. 分歧样本怎么处理

常见分歧：

- 文本违规但行为不印证：通常 mid，处置不宜直接 ban。
- 行为异常但文本很淡：看行为强度和时间关联，通常 mid 或 high。
- 外部联系方式：普通社交不违规，叠加交易/收益/色情才升档。
- 高风险主题但证据不完整：risk 可高，但 handling 要保守。

处理原则：宁可把重处罚交给人审，也不要让模型直接扩大 ban。

## 8. 面试表达

可以这样说：

“我没有把违规当成一个粗粒度标签，而是先和业务运营一起把 11 类主题拆成 47 个子主题，每个子主题写 low/mid/high rubric。标注员看到的是聊天证据、行为摘要和异常行为三段输入，独立标注 risk_level、final_judgment 和 handling_suggestion。对于 mid/low 分歧样本，我们用多数票和仲裁，并用 Fleiss Kappa、ordinal Krippendorff alpha 监控一致性。这样做的价值是把业务处置口径转成模型可学习的监督信号。”
